### 网页界面（推荐，不必用终端看结果）

在项目根目录执行：

```cmd
cd /d D:\people
run_web.cmd
```

或：

```cmd
python -m uvicorn people.webapp:app --host 127.0.0.1 --port 8770
```

**推荐**：双击 **`run_web.cmd`**，约 2 秒后会**自动打开浏览器**进入演练页；保持该黑窗口不关即后端在线。页面已**默认连接**本机 `http://127.0.0.1:8770` 的 API，用其它方式打开 `index.html` 时一般也无需再改配置。若你把后端改到别的端口，再编辑 `web/index.html` 里 **`people-api-origin`** 的 `content` 即可。选择场景、填 Key/Base URL/模型（可选）、点「开始运行」。**首次打开**会自动弹出 **「场景配置向导」**（也可随时点「场景配置向导」按钮）：在**不修改磁盘上 YAML 文件**的前提下，选择政策/产品演练类型、填写机关 `policy_context` 或企业 `issuer` 画像，并写本轮 **player_brief** 要点；点「应用到本次运行」后，这些字段会随 HTTP 请求**合并进场景**（人设、危机规则、cohort 等仍来自所选底稿 YAML）。运行中会**流式**展示：当前 tick、智能体调用、媒体报道倾向摘要、每 tick 末的**时间线展望**（今天～本周 → 约一月 → 约一年；若信任与支持度持续偏强，可出现**约两年、三年**情景，仅为推演非承诺）、**运行状态卡片内「账面现金 / 可用财力池 / 债务本金」快照条**（需场景开启 `operational_finance`；另有独立**财务曲线**避免与 0～1 宏观指标挤在同一纵轴）、以及**机构脉搏 / 合作报价**等细节。  
**指标曲线**：第一张图为宏观量；若场景开启 `extensions.kpi` / 扩散 / 资源池等，第二张图会显示 `kpi_*`、`diffusion_i_share`、资源池等（资源量走右侧 Y 轴）。**交流网络图**：跨主体沟通以有向边表示，**同起点→终点**的多次互动会**合并**为一条边，**线宽与 ×N** 表示次数；可点「适应画布」「重新布局」「物理」微调；悬停边看摘要。同页下方还可点 **「估计终局分布」** 做并行蒙特卡洛（与左侧同场景、种子与要点；**步骤说明与命令行**见 **系统说明.md §8.1**），查看分位数柱状图与 JSON 摘要。  
若场景 YAML 配置了 `crisis_rules` 且指标触线，演练会**暂停**并弹出「指挥台处置」——须**自行输入解决方案**后点「提交并继续」续跑；**引擎不会自动替你化解危机**（续跑依赖服务端内存中的 checkpoint，**进程重启后需重新开一局**）。  
说明：单次演练可能很长（如 `deepseek-reasoner`），请耐心等待，不要关页面。

### 真实感与「预测」边界（重要）

- 本工具输出的是**给定场景假设与指标下的情境推演**，用于沟通与资源压力测试；**不是**对真实销量、股价、案件结果或监管决定的**精准预测**。引擎已通过**更钝的宏观响应**（`realism` 中略降低单轮 LLM 对指标的边际冲击）与**提示词校准**（信息不完整、组织迟滞、条件句表述、去游戏/科幻口吻）尽量贴近可核对的真实语态。  
- 若要与现实对照，仍须以**线下数据、法务与业务事实**为准；可将 YAML 的 `player_brief`、指标初值与 `crisis_rules` 阈值尽量贴近你要复盘的真实区间。  
- **可选更「粘」的曲线**：场景 `realism.macro_inertia_blend`（默认 0，需自试）；**自动粗检异常跳变**：默认开启 `decision_support`，流式 `tick_end` 与 `run --dump-full-state` 会带通俗提示——详见 **系统说明 §6.1**。

### 地区情境检索（政策要贴地、开店要贴城）

- **目的**：先形成「**当地已有政策与执行习惯 + 中央上位框架 + 基层/街坊常态**」的合成摘要（可选拉一段 **中文维基** 作弱先验），再让你写的 **`player_brief`** 表示 **新政策或新开店动作**；人群与关键角色的反应会按提示词要求写成 **在旧常态上的增量**，而不是从零开始的真空世界。  
- **怎么开**：场景 YAML 里 **`regional_grounding.enabled: true`**，并填 **`region_label`**、**`mode`**（`llm_only` 或 `wikipedia_then_llm`）；商户/企业可加 **`business_scale`**（从街边店到集团）、**`business_sector_brief`**（如社区火锅店）。CLI：**`--regional-grounding`**、**`--region`**、**`--grounding-mode`**、**`--business-scale`** 等（**`ensemble` / `stability`** 同样支持）。Web：请求体字段 **`regional_grounding_enabled`**、**`regional_grounding_region`** 等与向导扩展对齐。  
- **可选「像搜索引擎一样」拉网页摘要**：设 **`regional_grounding.web_search_enabled: true`**，或 CLI **`--web-search`** / **`--web-search-provider duckduckgo|brave`**，或 HTTP **`regional_grounding_web_search`**、**`web_search_provider`**。默认走 DuckDuckGo（需 **`duckduckgo-search`**）。**国内公网**上 DuckDuckGo **经常连不上或结果空**，与用 **DeepSeek** 调 LLM **无关**（DeepSeek 只生成文本，不负责搜网页）；建议改用 **`--web-search-provider brave`** 并设 **`BRAVE_API_KEY`**，或关联网页检索、仅用 **`wikipedia_then_llm` / `llm_only`**。Brave 亦需环境变量 **`BRAVE_API_KEY`**（或 **`PEOPLE_BRAVE_API_KEY`**）。得到的是 **公开网页 snippet**，仍由快模型合成情境。  
- **诚实边界**：这是 **模型合成 + 可选开放百科 + 可选网页摘要**，**不是**官方法规检索或备案查询；涉法事项必须以贵方法务与权威来源为准。示例场景：**`scenarios/regional_hotpot_chengdu.yaml`**；机制见 **系统说明 §5.1**。

### 账面现金、税费、债务与手工调账

- **`operational_finance`** 现支持：按 **行业倍数** 与 **`issuer` 体量** 缩放 **每 tick 运营支出**；**写意营收 × 税负** 从现金扣税；**债务本金 + 年化利息（按约 26 tick/年）+ 分期还本**；政策沙盘下 **经济景气挂钩的写意税收入库**（增加财力池）。智能体 **`cash_delta_million`** 与交流 **`cost_million`** 仍会动账。  
- **暂停时**：网页弹窗 **「财务手工调账」** 或 API **`POST /api/finance/adjust-checkpoint`** 可改现金/财力/债务并写入叙事备注，返回 **新 checkpoint**。详见 **系统说明 §6.2**；小店示例仍见 **`regional_hotpot_chengdu.yaml`**。

### 给政府 / 企业信息部门的对照清单（简版）

正式环境落地前建议与 **系统说明 §19.2** 对照：**场景与 seed 归档**、**experiment_manifest（或 pause 包内副本）**、**所用模型路由**、是否启用 **性能预算退化**、对外部真值做 **validate** 时是否写清数据来源。涉密与个人信息进模型、等保关基对齐等事项见 **系统说明 §1.5**（本软件不替代贵方保密制度与测评流程）。

### 页顶「仅评估本草案（内部）」

- **随时可用**：不必等危机暂停；**流式主演练进行中**也可在网页点「仅评估」（独立请求，不中断 SSE）。`POST /api/plan/evaluate` 在不传 `checkpoint_token` 时按**当前所选 YAML + 向导覆盖 + 指挥台要点**构造 **tick 0** 态势做专业转化与**内部桌面研判**。  
- **默认不绑定快照**：网页上**默认不勾选**「绑定当前 checkpoint」；只有勾选且存在暂停 token 时，才把**当时**舆情/账面进度纳入上下文。  
- **资源粗估与综合判断**：引擎会把 **plan_evaluate_live_context** 注入 `decision_context`（实盘宏观、账面/财力/债务、人力与政治资本池、cohort 快照、叙事尾部，外加 YAML 财务参数、记忆、决策层指令、地区情境、机构与合作等）。专业转化与桌面研判均须**统筹**这些信息后再写可执行性与资源区间；若与草案冲突须在结论中写明。页面上**账面现金 / 负债**以**沙盘记账数值**为主（演练用 proxy）；请在 YAML 的 `operational_finance` 写明贴近真实体量的初值；若仍为模板默认（如现金 800、负债 0），引擎可按 `issuer` 体量档位生成写意初值以便自洽。
- **评估后的「确认执行」**（网页按钮）：仅评估**不会**改沙盘；点 **「确认执行本方案」** 后调用 `POST /api/plan/confirm-execute`，把结论写入 **`decision_layer_active_summary`**、**公司记忆**与叙事，并对**宏观与账面/财力**做一轮克制边际调整；同时根据转化/桌面时间线文案登记**未来某一 tick**的「约定检视」节点，推进到该 tick 时在叙事与流式日志中输出。**绑定 checkpoint** 时会生成**新 checkpoint** 供续跑；未绑定时则排队到**下一次「开始运行」**在 tick0 写入（请求体 `internal_plan_commit`）。多次评估的草案会进入下拉**候选列表**便于切换；「不采纳，另写一套」清空当前草案框以便重写。

### 白话处置 → 专业方案（产品 / 政策通用）

暂停时**可以用口语**写「你想怎么办」。续跑开始后，系统会用 **FAST 模型**先做一步「专业转化」：

- **产品端**：把白话理解成意图，再写成执行层可分工的表述（公关口径边界、法务合规、客服与渠道、供应链与交付、财务与预算等择要），并给出 **可执行性判断**（含前提与风险）。转化结果会写入叙事与后续各智能体上下文（演练生成，**不是**真实法务意见或公司决议）。
- **创业公司端**（`issuer.archetype: startup`）：白话转化侧重**创始人盯盘、砍范围、外包律师/兼职财务、社群与增长承接**；关系图里的基线部门链会换成精简链路（创始人—产品小队—增长运营—财务/法务外包等），指挥中心显示为「创业公司指挥中心」。示例：`scenarios/startup_example.yaml`（`product_launch.yaml` 里若已写 `startup` 同样生效）。
- **政策端**：把白话理解成政策与稳控关切，并**映射到常见行政流程顺序**（议题与目标 → 调研与部门会商 → 合法性/合规把关 → 社会稳定与舆情风险评估 → 征求意见或听证等程序（如适用）→ 集体审议决策 → 公布与解读 → 执行与督查评估），同样给出可推进性与前提。说明：真实世界以当地法定程序与集体决策为准，本工具只做**压力测试式推演**。

流式界面里会出现事件 **`resolution_digest`**（意图理解、可执行性、专业方案节选、政策流程映射）。

### 何时暂停要你出方案？（推荐：指标阈值 `crisis_rules`）

- **默认不再**按「调用满 N 次 LLM」暂停： **`risk_milestone_primary_calls` 默认为 `0`（关闭）**。其它 tick 由引擎内的智能体、部门交流链与公司记忆照常推演，**不强制**指挥台输入。  
- **需要过线才暂停**时，请在场景 YAML 里配置 **`crisis_rules`**：对 `rumor_level`、`unrest`、`policy_support` 等设 `when_all` 条件（与 `triggers` 类似），触线后本 tick 结束会暂停并等你提交方案。示例见 `product_launch.yaml`、`startup_example.yaml`。  
- **可选**：若仍希望「累计关键推演满 N 次必停一次」，再把 **`risk_milestone_primary_calls`** 设为大于 0 的整数；与 `crisis_rules` 同时存在时，**同一 tick 末优先 `crisis_rules` 暂停**，内部风险盘点顺延。  
- **续跑**时顺序为：对**每一套**指挥台方案依次 **白话 → 专业转化**，再对每一套做 **公司内部桌面可行性研判**（事件 **`internal_feasibility_digest`**，带 **`plan_index`**）；研判结果中应包含 **`why_not_feasible`**（不可行/存疑点条目）。这是 **内部战研/职能桌面推演**，与真实情况 **可能一致也可能不一致**。  
- **默认不会「研判完就立刻跑下一 tick」**：场景里 **`confirm_after_desk_feasibility`** 默认为 **`true`**。研判结束后会再暂停一次（**待确认**），须在前端点 **「确认继续演练」** 后才会进入后续 tick；若设为 **`false`**，则研判结束后直接继续 tick。  
- **多套方案**：在文本框中用**单独一行 `---`** 分隔每一段方案；接口亦支持 JSON 字段 **`user_solutions`**（字符串数组）。可选在「待确认」弹窗里 **选用主执行线**（写入叙事备注）。  
- **公司内部记忆**：每个 tick 在关键智能体表态结束后，用快模型把本轮要点**压缩进状态**（`company_memory_synthesis`），并注入后续 **风险盘点、桌面研判、白话转化** 与 **关键智能体** 的 `decision_context`，用于减少重复踩坑（仍受模型与 token 限制，非完美记忆）。  
- **产品侧舆论与指标**：场景 **`problem_salience`**（0～1，议题/需求刚需程度）与发行方 **`issuer.brand_equity`** 会共同调制 **关键智能体产出** 作用到 `policy_support`/谣言/动荡等时的边际效果：刚需高则负面冲击更易放大；品牌资产高则对**负面支持度、不利谣言、动荡**等有一定缓冲（类似「报道差但品牌强仍有人买单」的粗粒度 proxy，非个体购买预测）。

- 盘点快照仅在当次续跑研判中使用，随后从状态中清除。

### 把软件交给政策方时的简要建议（可随 U 盘打印一页）

1. **先约定演练边界**：本工具输出用于研判舆情与执行摩擦，**不替代**合法性审查、风险评估、集体决策与正式发布程序。  
2. **对照熟悉的决策链条**（通俗理解《重大行政决策程序暂行条例》等精神）：**公众参与、专家论证、风险评估、合法性审查、集体讨论决定**等环节，在演练里被抽象为「部门交流网络 + 指标 + 暂停处置」；可在 YAML 里用 `crisis_rules` 强制「必须指挥台输入方案」的节点。  
3. **使用方式建议**：`player_brief` 写清拟推措施要点与约束；`policy_context` 填行政层级与地名、民风与媒体环境；重大事项应安排**真人法务 / 法制 / 财政**等线下把关，再把结论回填为下一轮 `player_brief` 或处置输入。  
4. **与产品端的区别**：产品侧侧重市场、监管、供应链与现金；政策侧侧重程序节奏、条线与基层执行——两者在演练里都走「白话指挥 → 专业转化 → 多主体交流」同一套机制，便于统一培训。

**场景里可配的扩展字段（示例见 `product_launch.yaml`、`startup_example.yaml`、`policy_county_example.yaml`）**：**`crisis_rules`**（推荐用于「风险/问题过线才暂停」）、**`risk_milestone_primary_calls`**（默认 `0`；>0 为可选的「满 N 次关键 LLM 暂停」）、**`confirm_after_desk_feasibility`**、**`problem_salience`** 等；另有 `operational_finance`、`institutions`、`cooperations`。

**若提示「端口只允许使用一次」**：上次服务还在占用端口。可双击 **`结束端口占用.cmd`**（默认杀 8770；要杀 8780 可在 cmd 里执行 `结束端口占用.cmd 8780`），或把 `run_web.cmd` 里的 `PORT` 改成别的数（如 8780）。

配置 API（至少一种）：

- `OPENAI_API_KEY`（或兼容网关 + `OPENAI_BASE_URL`）
- 或 `ANTHROPIC_API_KEY`
- 或 `GOOGLE_API_KEY` / `GEMINI_API_KEY`

### 用 DeepSeek（推荐最强：`deepseek-reasoner`）

DeepSeek 与 OpenAI SDK 兼容，本项目用 **同一套环境变量** 指向 DeepSeek：

**CMD（可先双击改好密钥后，在 cmd 里 `call 配置示例-DeepSeek.cmd`）**

```cmd
set OPENAI_API_KEY=你的DeepSeek密钥
set OPENAI_BASE_URL=https://api.deepseek.com/v1
set PEOPLE_MODEL_PRIMARY=openai:deepseek-reasoner
set PEOPLE_MODEL_FAST=openai:deepseek-chat
```

说明：

- **密钥**在 [DeepSeek 开放平台](https://platform.deepseek.com/) 创建；**Base URL 必须带 `/v1`**。
- **`deepseek-reasoner`**：推理模型，适合沙盘里「关键人设」；响应较慢，超时已自动放宽。
- **`deepseek-chat`**：便宜快，适合 `cohort_llm` 打开时的批量更新；若暂不用 cohort LLM，FAST 也可先与 PRIMARY 相同。

也可命令行单次指定（不改环境变量）：

```cmd
python main.py run --scenario scenarios\default.yaml --primary openai:deepseek-reasoner --fast openai:deepseek-chat
```

（仍需已设置 `OPENAI_API_KEY` 与 `OPENAI_BASE_URL`。）

### 用 Google Gemini

在 [Google AI Studio](https://aistudio.google.com/apikey) 创建 API Key，使用环境变量 **`GOOGLE_API_KEY`** 或 **`GEMINI_API_KEY`**（二选一）。模型路由前缀必须是 **`google:`** 或 **`gemini:`**。

**CMD（见根目录 `配置示例-Gemini.cmd`）**

```cmd
set GOOGLE_API_KEY=你的Gemini密钥
set PEOPLE_MODEL_PRIMARY=google:gemini-2.0-flash
set PEOPLE_MODEL_FAST=google:gemini-2.0-flash
```

### 用 Anthropic Claude

在 [Anthropic Console](https://console.anthropic.com/) 创建密钥，设置 **`ANTHROPIC_API_KEY`**，路由形如 **`anthropic:claude-sonnet-4-20250514`**。详见根目录 **`配置示例-Claude.cmd`**。

可选：指定其它厂商模型（覆盖默认）

```powershell
$env:PEOPLE_MODEL_PRIMARY = "anthropic:claude-sonnet-4-20250514"
```

## 2. 两种命令

| 命令 | 用途 |
|------|------|
| `run` | **跑一条**时间线：看叙事与终局指标 |
| `ensemble` | **同一设定跑多遍**（不同随机种子）：看结局分布/频率（模型内估计，不是真实世界概率） |

---

## 3. 单次演练 `run`

```powershell
python main.py run --scenario scenarios\default.yaml
python main.py run --scenario scenarios\policy_county_example.yaml
python main.py run --scenario scenarios\society_export_software.yaml --show-budget
```

常用参数：

| 参数 | 说明 |
|------|------|
| `--scenario` | 场景 YAML 路径 |
| `--seed` | 随机种子（换种子 = 另一种「世界线」） |
| `--brief` | 临时覆盖「你下达的政策/产品说明」 |
| `--brief-file` | 从 UTF-8 文件读长文说明（优先于 `--brief`） |
| `--reference-file` | 脱敏**真实案例** UTF-8 文本，供模型对照机制（不写细节当定论） |
| `--exercise` | `policy` 或 `product`，覆盖 YAML 里的类型 |
| `--primary` / `--fast` | 模型，如 `openai:gpt-4o`、`google:gemini-2.0-flash` |
| `--show-budget` | 打印每轮大约调用多少次「人设 LLM」 |

---

## 4. 批量并行 `ensemble`

```powershell
python main.py ensemble --scenario scenarios\product_launch.yaml --runs 24 --workers 4 --dump-json out.json
```

| 参数 | 说明 |
|------|------|
| `--runs` | 跑多少条独立世界线 |
| `--workers` | 并行进程数（Windows 下别太大） |
| `--threshold-key` / `--threshold` | 例如 `policy_support` 是否 ≥ 0.55 的频率 |
| `--dump-json` | 完整结果落盘，便于做表、归档 |

`--brief-file`、`--reference-file`、`--exercise` 与 `run` 相同。

## 5. 改场景 = 改 YAML（核心工作）

复制 `scenarios\` 里任一场景，改文件名后编辑。最重要字段：

### 政策向 `exercise_type: policy`

- **`player_brief`**：条文要点、过渡期、执法口径、宣传重点。
- **`policy_context`**：`admin_level`（`central` / `province` / `city` / `county`）、`jurisdiction_name`、`local_norms_brief`（民风人情）、`media_environment_brief`。
- **`cohorts`**：大众结构；每条可加 **`class_layer`**：`lower` / `middle` / `upper` / `mixed`（基层/中产/高层 proxy）。
- **`key_actors`** 或 **`personas`**：关键角色与人设名册（大规模用人设 + 池化抽样，见 README）。

### 产品向 `exercise_type: product`

- **`player_brief`**：产品形态、定价、渠道、合规与传播。
- **`issuer`**：`archetype`（`megacorp` / `large_group` / `sme` / `startup`）、`brand_equity`（0～1）、`reputation_brief`、`supply_chain_position`。
- **`market_competitors`（可选，推荐）**：**多家**竞品/替代方案的结构化列表（`id`、`name`、`brief`、可选 `estimated_market_share_proxy`、可选 `linked_key_actor_id` 对齐 `key_actors`）。写入后进入 `decision_context`，关键智能体与社交回合须承认竞争格局，避免「全场只有你一家公司」。示例见 `product_launch.yaml`。
- **`key_actors`**：为竞品侧至少配一名 **`categories: [competitor]`** 的发言人（如公关负责人），可与 `market_competitors[].linked_key_actor_id` 对齐。
- **`simulation`**：`product_kind`、`markets_active` 等。
- **`cohorts` + `class_layer`**：用户与公众分层。
- **`personas` + `triggers` + `realism`**：见 `society_export_software.yaml` 完整示例。

### 真实案例

- 在 YAML 里写 **`reference_cases_brief`**，或命令行 **`--reference-file 某.txt`**（UTF-8，脱敏）。

## 6. 指标怎么读（给汇报用）

- **`policy_support`**：政策盘 = 对政策/施政接受度；产品盘 = 对发行方动作的支持/好感（含购买意愿 proxy）。
- **`unrest`**：争议、对抗、街头化风险叙事强度 proxy。
- **`issuer_trust_proxy`**：主体/品牌信任慢变量。
- **`supply_chain_stress`**：交付与外部冲击叙事压力 proxy。
- **`sentiment` / `rumor_level` / `economy_index`**：舆情与景气 proxy。

说明：这是 **LLM + 简化动力学** 的压力测试，**不能**替代统计调查、法务与监管结论。

## 8. 遇到问题

- **中文乱码**：终端编码设为 UTF-8，或主要看 `--dump-json` 文件（建议 UTF-8 保存）。
- **并行报错**：`ensemble` 先试 `--workers 1`。
- **费用高**：减少 `ticks`、`key_actors`/`llm_each_tick` 人数、或 `pooled_llm_calls_per_tick`，或换便宜 `--primary` 做初筛。

更细的机制说明见项目根目录 **`README.md`**。
